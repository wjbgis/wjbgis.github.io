Git学习笔记

发表于 2019-08-01

Git命令行操作

1. 本地库初始化

  • 在本地库打开Git Bash (windows下：右键->Git Bash Here)
  • 命令：git init
  • 效果：本地库产生一个.git文件夹（隐藏文件夹）
  • 注意：.git文件夹中存储的是本地库相关的子目录和文件，不要随意删除、修改

设置签名

  • 签名形式：

    ​ 用户名：wjb Email：wjbgis@foxmail.com

  • 作用：区分不同开发人员的身份，不允许没有签名

  • 辨析：这里设置的签名和登录远程库的用户名、密码没有关系

  • 级别：项目（仓库）级别、系统级别。（项目级别优先级大于系统级别）

  • 命令：见下方代码块

基本操作

  • 查看工作区、暂存区的状态：git status

  • 将工作区新建/修改的内容添加到暂存区：git add [file name]

  • 将暂存区的内容提交到本地库：git commit -m "do sth" [file name]

  • 查看日志：git log git log --pretty=oneline git log --oneline

4.前进后退

4.1 版本前后移动

  • 本质：HEAD指针的移动
  • 基于索引值的操作（推荐）：git reset --hard [局部索引值]
  • 使用^符号，只能后退：git --hard HEAD^，注：n个^表示后退n个版本
  • 使用符号，只能后退：`git–hard HEADn`，注：表示后退n个版本

4.2 reset命令的三个参数对比

  • --soft参数

    ​ 仅在本地库移动HEAD指针

  • --mixed参数

    ​ 在本地库移动HEAD指针、重置暂存区

  • --hard参数

    ​ 在本地库移动指针、重置暂存区、重置工作区

4.3 删除文件找回

  • 前提：删除前，文件存在时的状态提交到了本地库
  • 命令：git reset --hard [指针位置]
    • 删除操作已经提交到本地库：指针位置指向历史记录
    • 删除操作尚未提交到本地库：指针位置使用HEAD

4.4 比较文件差异

  • git diff [文件名]：将工作区和暂存区中的文件比较

  • git diff [本地库中历史版本][文件名]：

    ​ 将工作区中的文件和本地历史记录比较

  • 不带文件名比较多个文件

5.分支管理

  • 分支：在版本控制过程中，使用多条线同时推进多个任务。

  • 分支的好处：

    • 同时推进多个功能开发，提高开发效率
    • 某一个分支出现问题，不会对其他分支产生影响

  • 分支的种类

    1. 主干分支：master

      负责管理正在运行的生产环境代码，永远保持与生产环境代码完全一致。

    2. 开发分支：develope

      负责管理正在开发过程中的代码，一般为最新代码

    3. bug修理分支：hotfix

      负责管理生产环境中需要紧急修复的代码，从master中分出，修改测试完毕后，并回主干分支，生产环境稳定后可视情况删除

    4. 预发布分支：release

      较大版本上线前，会从开发分支中分出准生产分支进行最后的集成测试，该版本上线后会合并到master分支，生产环境稳定后可视情况删除

    5. 功能分支：feature

      为了不影响短周期的开工作，一般把中长期的开发模块从develop分支中独立出来，开发完成后合并到develop分支

  • 分支操作：

    • 创建分支：git branch [分支名]

    • 查看分支：git branch -v

    • 切换分支：git checkout [分支名]

    • 合并分支：

      1. 切换到接受修改的分支上git checkout [被合并分支名]

      2. 第二步：执行merge命令 git merge [有新内容的分支]

    • 解决冲突：

  1. 编辑文件，删除特殊符号
  2. 把文件修改至满意程度，保存退出
  3. git add [文件名]
  4. git commit -m "日志信息" 注意：这里千万不要带文件名！

6.Github

  • 创建远程库地址别名：git remote add [别名] [远程地址]

  • 查看所有远程地址别名：git remote -v

  • 推送：git push [别名] [分支名]

  • 克隆：git clone [远程地址] 效果如下：

    • 完整的把远程库下载到本地
    • 自动创建origin远程地址别名
    • 出啊石化本地仓库

  • 团队成员邀请：

  • 拉取

    • pull = fetch + merge
    • git fetch [远程库地址别名] [远程分支名]
    • git merge [远程库地址别名/远程分支名]
    • git pull [远程库地址别名] [远程分支名]
    • 冲突问题

    • 如果本地库和远程库版本不一致（远程库版本高于本地库版本），本地库做修改后提交时会产生冲突

    • 此时需要先把远程库用pull命令拉取下来，

    • 拉取下来的文件和本地已修改的文件必然有冲突，将冲突内容修改至满意程度后

    • 再分别执行git add [File name] git commit -m "do sth" git push [远程库别名][分支名]

    • 注意，此时的commit不能带文件名
