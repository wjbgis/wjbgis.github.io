Docker学习笔记

一、为什么使用Docker

1. 环境配置的难题

软件开发最大的麻烦之一就是环境配置，用户计算机环境不同，所做的配置可能也不同。如安装java应用需要有响应版本的java环境支持，还要有各种依赖的支持；C#应用需要相应版本的.NetFramwork的支持。

能不能从根本上解决问题，软件可以带环境安装。也就是说可以把原始运行环境一块搬过来。

2. 虚拟机

虚拟机是带环境安装的一种解决方案，它可以在一种操作系统里面运行另一种操作系统。用户可以通过虚拟机还原软件的原始环境。

但虚拟机也有一定缺点：

  • 资源占用多
  • 冗余步骤多
  • 启动慢

3. Linux容器

由于虚拟机存在这些缺点，Linux 发展出了另一种虚拟化技术：Linux 容器（Linux Containers，缩写为 LXC）。

Linux 容器不是模拟一个完整的操作系统，而是对进程进行隔离。或者说，在正常进程的外面套了一个保护层。对于容器里面的进程来说，它接触到的各种资源都是虚拟的，从而实现与底层系统的隔离。

由于容器是进程级别的，相比虚拟机有很多优势。

  • 体积小
  • 资源占用少
  • 启动快

4. Docker

Docker 属于 Linux 容器的一种封装，提供简单易用的容器使用接口。它是目前最流行的 Linux 容器解决方案。

Docker 将应用程序与该程序的依赖，打包在一个文件里面。运行这个文件，就会生成一个虚拟容器。程序在这个虚拟容器里运行，就好像在真实的物理机上运行一样。有了 Docker，就不用担心环境问题。

总体来说，Docker 的接口相当简单，用户可以方便地创建和使用容器，把自己的应用放入容器。容器还可以进行版本管理、复制、分享、修改，就像管理普通的代码一样。

Docker的用途：

（1）提供一次性的环境。比如，本地测试他人的软件、持续集成的时候提供单元测试和构建的环境。

（2）提供弹性的云服务。因为 Docker 容器可以随开随关，很适合动态扩容和缩容。

（3）组建微服务架构。通过多个容器，一台机器可以跑多个服务，因此在本机就可以模拟出微服务架构。

Docker 把应用程序及其依赖，打包在 image 文件里面。只有通过这个文件，才能生成 Docker 容器。image 文件可以看作是容器的模板。Docker 根据 image 文件生成容器的实例。同一个 image 文件，可以生成多个同时运行的容器实例。

二、Docker安装与使用（Centos 7）

1. 安装与启动

安装：yum install docker

启动：systemctl start docker

开机启动：systemctl eable docker

2. Docker命令

搜索镜像：docker search 镜像名

拉取：docker pull 镜像名:tag tag可选，表示版本，默认是latest

查看本地所有镜像：docker images

删除指定的本地镜像：docker rmi images_id

3. 容器操作

运行镜像：docker run --mytomcat -d tomcat -d表示后台运行

运行时进行端口映射：docker run -d -p 8888:8080 tomcat 将主机端口8888映射到docker的8080端口

查看运行中的容器：docker ps

查看运行中的所有容器：docker ps -a

停止容器：docker stop container_id

启动容器：docker start container_id 启动已经运行（run）但被stop的容器

删除容器：docker rm mysql01

查看启动日志：docker logs container_id

4. Docker启动MySQL

最好去官网搜索mysql容器然后查看相应说明

docker run -p 3306:3306 --name mysql01 -e MYSQL_ROOT_PASSWORD=123456 -d mysql
